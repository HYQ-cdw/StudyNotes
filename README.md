## 二、详细过程说明——毕设用户认证模块开发

### 2.1 需求分析

用户认证模块需要支持用户注册、登录、Token刷新等功能，采用JWT实现无状态认证。

### 2.2 实现步骤

**Step 1：环境搭建**
在项目中引入 `jsonwebtoken` 和 `bcryptjs` 依赖包，配置JWT密钥和过期时间。

**Step 2：用户注册功能**
```javascript
// 密码加密存储
const salt = await bcrypt.genSalt(10);
const hashedPassword = await bcrypt.hash(password, salt);

// 用户数据入库
const newUser = await User.create({ username, email, password: hashedPassword });
```

**Step 3：用户登录功能**
```javascript
// 密码校验
const isMatch = await bcrypt.compare(password, user.password);
if (!isMatch) throw new Error('密码错误');

// 生成Token
const token = jwt.sign({ userId: user._id }, SECRET_KEY, { expiresIn: '7d' });
```

**Step 4：Token验证中间件**
编写Express中间件，解析请求头中的Authorization字段，验证Token有效性并挂载用户信息。

### 2.3 核心代码结构

```
routes/auth.js       # 认证路由
middleware/auth.js   # 鉴权中间件
models/User.js       # 用户数据模型
utils/jwt.js         # Token工具函数
```

## 三、遇到的问题及解决方法

### 问题1：JWT Token过期导致用户频繁登出

**问题描述**：Token设置过期时间为1小时，用户在使用过程中频繁需要重新登录，体验很差。

**解决方法**：引入Refresh Token机制，Access Token有效期设为15分钟，Refresh Token有效期设为7天。前端自动在Access Token即将过期时使用Refresh Token获取新的Access Token。

### 问题2：密码加密性能问题

**问题描述**：bcrypt加密强度设为12时，登录验证耗时较长，高并发下影响系统响应。

**解决方法**：将加密强度调整为10，在安全性和性能之间取得平衡；同时对高频访问接口添加缓存层。

### 问题3：Swagger文档与实际接口不同步

**问题描述**：代码更新后忘记手动同步Swagger文档，导致文档与实际接口不一致。

**解决方法**：使用swagger-autogen自动根据代码注释和路由配置生成文档，确保文档始终与代码保持同步。
